Run exactly these commands from the repo root (growth-signal-outbound, main):

  bash scripts/cloud-send.sh

Do NOT run `python run.py composio-send` directly — use the script above.
Queue file is `send_queue.json` at repo root (NOT output/send_queue.json).

COMPOSIO_API_KEY must be in Cloud Agent secrets.

After the script finishes, read send_queue.json and report:
- emails sent this run (status changed to "sent")
- still pending + next scheduled_at
- if all sent: "All outbound emails complete."

If send_queue.json is missing after clone, stop and report — do not rebuild the queue.
